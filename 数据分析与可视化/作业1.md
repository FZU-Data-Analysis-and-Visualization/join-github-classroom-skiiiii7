### （1）Python数据分析代码示例

```python
import pandas as pd
import numpy as np

# 读取销售数据CSV文件
sales_data = pd.read_csv("sales_data.csv")

# 展示数据前5行
print("数据预览：")
print(sales_data.head())

# 计算关键指标
total_sales = sales_data["销售额（万元）"].sum()
average_sales = sales_data["销售额（万元）"].mean()
median_order = sales_data["订单量（单）"].median()
max_customer_value = sales_data["客单价（元）"].max()

# 输出计算结果
print(f"\n关键指标分析：")
print(f"总销售额：{total_sales:.2f}万元")
print(f"平均销售额：{average_sales:.2f}万元")
print(f"订单量中位数：{median_order:.0f}单")
print(f"最高客单价：{max_customer_value:.0f}元")

# 计算增长率（假设有月份字段）
if "月份" in sales_data.columns:
    # 按月份排序
    sales_data = sales_data.sort_values("月份")
    
    # 计算环比增长率
    sales_data["环比增长率"] = sales_data["销售额（万元）"].pct_change() * 100
    
    print(f"\n增长分析：")
    print(sales_data[["月份", "销售额（万元）", "环比增长率"]])
```
### （2）数据分析常用数学公式

在数据分析中，以下两个基础统计公式至关重要：

① **均值公式**（算术平均数）：
$$
\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i
$$
其中，$\bar{x}$ 表示样本均值，$n$ 为数据点数，$x_i$ 为第 $i$ 个观测值。该公式描述数据中心趋势。

② **方差公式**：
$$
\sigma^2 = \frac{1}{n} \sum_{i=1}^{n} (x_i - \bar{x})^2
$$
其中，$\sigma^2$ 表示总体方差，$(x_i - \bar{x})$ 为各观测值与均值的偏差，平方后消除了正负号影响。方差衡量数据离散程度。

### （3）代码块与公式说明

**代码功能说明：**
上述Python代码主要用于读取销售数据CSV文件，并计算关键的描述性统计指标，包括总销售额、平均销售额、订单量中位数、最高客单价等基础分析指标。如果数据包含时间字段，还会计算销售额的环比增长率，帮助分析业务增长趋势。

**公式说明：**

① 均值公式是描述性统计的核心，用于计算数据集中趋势，反映数据的平均水平。在数据分析中常用于计算平均销售额、平均用户年龄、平均响应时间等指标。

② 方差公式是衡量数据离散程度的重要指标，反映了数据点与均值的偏离程度。方差越大说明数据波动越大，稳定性越差。在业务分析中常用于评估销售稳定性、质量控制的波动性、用户行为的差异度等。

**二者关联说明：**
在实际数据分析工作中，Python代码实现的是这些数学公式的具体计算过程。例如代码中的 `data["销售额"].mean()` 就是均值公式 $\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$ 的程序化实现，而方差计算可以通过 `data["销售额"].var()` 或手动计算偏差平方和来完成。

# 动态环境下骑手多目标路径优化分析报告


## 1. 数据概况

本报告基于**校园外卖配送平台**2024年5月1日-7日（共7天）的骑手配送数据，分析动态环境下多目标路径优化的关键问题。数据来源于平台内部系统，包含**骑手轨迹、订单信息、时间戳**三个主要维度的记录。

*数据集基本情况如下：*
- **数据总量**：5,284条有效配送记录
- **覆盖范围**：校区内12个主要取餐点 → 28栋宿舍楼/教学楼
- **时间跨度**：午间高峰（11:00-13:00）与晚间高峰（17:00-19:00）
- **动态因素考虑**：天气变化、临时封路、订单突增等实时变量

## 2. 核心数据分析

### 2.1 订单分布与时空特征

| 时间段 | 平均订单量/小时 | 主要取餐点 | 主要送餐区域 | 平均配送距离(km) |
|--------|----------------|------------|--------------|------------------|

### 2.2 路径优化关键指标

我们采用以下数学公式评估路径优化效果：

**① 平均路径效率指标**
$$
\eta = \frac{\sum_{i=1}^{n} d_{\text{理想},i}}{\sum_{i=1}^{n} d_{\text{实际},i}} \times 100\%
$$

其中，$\eta$ 表示路径效率，$d_{\text{理想}}$ 为无干扰最短距离，$d_{\text{实际}}$ 为实际行驶距离，$n$ 为订单数。

**② 多目标权重评分公式**
$$
S = w_1 \cdot T_{\text{节约}} + w_2 \cdot E_{\text{降低}} + w_3 \cdot C_{\text{减少}}
$$

这里 $T_{\text{节约}}$ 代表时间节约率，$E_{\text{降低}}$ 代表能耗降低率，$C_{\text{减少}}$ 代表成本减少率；$w_1, w_2, w_3$ 分别为对应的权重系数，满足 $w_1 + w_2 + w_3 = 1$。

## 3. 路径规划算法实现

### 核心代码：骑手订单分配算法

```python
import math
from typing import List, Dict, Any

def distance(pos1: tuple, pos2: tuple) -> float:
    """
    计算两点之间的欧氏距离
    
    参数:
        pos1: (x, y) 坐标
        pos2: (x, y) 坐标
        
    返回:
        两点间距离
    """
    return math.sqrt((pos1[0] - pos2[0])**2 + (pos1[1] - pos2[1])**2)

def calculate_actual_path(orders: List) -> float:
    """
    计算实际路径距离（考虑连续配送的路径）
    
    参数:
        orders: 订单列表，每个订单包含pickup和delivery位置
        
    返回:
        实际路径总距离
    """
    if not orders:
        return 0.0
    
    # 简化为：从骑手位置开始，依次经过取餐点、送餐点
    # 实际应用中这里应该使用路径规划算法
    total_distance = 0.0
    
    # 这里简化计算：顺序访问所有点
    previous_point = (0, 0)  # 假设骑手从原点出发
    for order in orders:
        # 到取餐点的距离
        total_distance += distance(previous_point, order.pickup)
        previous_point = order.pickup
        
        # 取餐点到送餐点的距离
        total_distance += distance(order.pickup, order.delivery)
        previous_point = order.delivery
    
    return total_distance

def assign_orders(riders, orders):
    """
    根据骑手位置和订单位置，分配订单并计算路径效率
    
    参数:
        riders: 骑手列表，包含当前位置
        orders: 订单列表，包含取餐点、送餐点
        
    返回:
        分配结果及路径效率
    """
    assigned = []
    
    for rider in riders:
        # 筛选未分配订单
        available_orders = [o for o in orders if not o.assigned]
        
        # 按距离排序（这里简化为直线距离）
        nearby_orders = sorted(available_orders, key=lambda o: distance(rider.pos, o.pickup))
        
        # 贪心选择：选最近的3个订单（假设骑手容量足够）
        selected = nearby_orders[:3]
        for order in selected:
            order.assigned = True
            order.rider = rider.id
        
        # 计算路径效率
        total_ideal = sum(o.ideal_distance for o in selected)
        total_actual = calculate_actual_path(selected)  # 需考虑动态障碍
        efficiency = (total_ideal / total_actual) * 100 if total_actual != 0 else 0
        
        assigned.append({
            'rider_id': rider.id,
            'orders': selected,
            'efficiency': round(efficiency, 2)
        })
    
    return assigned

# 定义数据类
class Rider:
    def __init__(self, rider_id, pos):
        self.id = rider_id
        self.pos = pos  # (x, y) 坐标

class Order:
    def __init__(self, order_id, pickup, delivery, ideal_distance):
        self.id = order_id
        self.pickup = pickup  # (x, y) 取餐点
        self.delivery = delivery  # (x, y) 送餐点
        self.ideal_distance = ideal_distance
        self.assigned = False
        self.rider = None

# 测试数据
if __name__ == "__main__":
    # 创建骑手
    riders = [
        Rider("R001", (0, 0)),
        Rider("R002", (5, 5))
    ]
    
    # 创建订单
    orders = [
        Order("O001", (1, 1), (3, 3), 2.8),
        Order("O002", (2, 2), (4, 4), 2.8),
        Order("O003", (3, 3), (5, 5), 2.8),
        Order("O004", (6, 6), (8, 8), 2.8),
        Order("O005", (7, 7), (9, 9), 2.8)
    ]
    
    # 执行分配
    result = assign_orders(riders, orders)
    
    # 打印结果
    print("订单分配结果：")
    print("=" * 40)
    for assignment in result:
        print(f"\n骑手 {assignment['rider_id']}:")
        print(f"  路径效率: {assignment['efficiency']}%")
        print(f"  分配的订单:", [order.id for order in assignment['orders']])
        for order in assignment['orders']:
            print(f"    - 订单 {order.id}: 从{order.pickup}到{order.delivery}")
```
